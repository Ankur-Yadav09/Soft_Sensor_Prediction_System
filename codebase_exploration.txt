  ---
  Architecture Overview

  This is a single-file Streamlit web app (app.py, ~2200 lines) for industrial
  soft sensor prediction using a Denoising Autoencoder (DAE) neural network.

  ---
  Neural Network: IndustrialDAE

  A multi-task Denoising Autoencoder with 3 components:

  Corrupted X (masked) ──→ Encoder ──→ Latent State (bottleneck)
                                            │
                                ┌───────────┴───────────┐
                                ↓                       ↓
                             Decoder                Predictor
                         (Reconstruct X)          (Predict Y/KPIs)
                         MSE Loss                 Huber Loss

  - Encoder: input → 128 → 64 → latent_dim (BatchNorm + ReLU + Dropout)
  - Decoder: Reconstructs all input sensor features (denoising)
  - Predictor: Directly predicts KPI targets from latent state
  - Combined loss: recon_loss + weight × pred_loss

  The key idea: by learning to reconstruct corrupted/masked sensor data, the
  model learns a robust latent representation that generalizes better to KPI
  prediction.

  Full Data Pipeline

  Excel Upload
      ↓
  SQLite (dataset versioning)
      ↓
  Preprocessing (imputation → outlier capping → feature selection →
  StandardScaler)
      ↓
  Train/Test Split (80/20)
      ↓
  Training Loop (masked inputs → DAE → dual loss → Adam optimizer)
      ↓
  Auto-save: model.pth + scaler_x.pkl + scaler_y.pkl + columns.pkl +
  metadata.pkl

  Evaluation (test set metrics + visualizations)
      ↓
  What-If Analysis (feature sweep → KPI sensitivity)

  ---
  Persistence Layer

  - dashboard.db — SQLite database storing uploaded datasets as Parquet blobs
  - saved_models/<run_id>/ — Each training run saves:
    - model.pth — PyTorch weights
    - scaler_x.pkl / scaler_y.pkl — fitted StandardScalers
    - columns.pkl — feature name mappings
    - metadata.pkl — hyperparams, timestamps, metrics

  ---
  Sample Data

  Data_DAE.xlsx — an industrial sensor dataset with ~41 input sensor features
  (X) and ~5 target KPI features (Y). Two trained model runs already exist in
  saved_models/.

  ---
  Key ML Concepts

  1. Denoising Autoencoder: Random feature masking during training forces the
  model to learn robust inter-sensor correlations
  2. Multi-task learning: Joint reconstruction + KPI prediction from the same
  latent space
  3. Masking ratio: What fraction of input features are zeroed out per sample
  during training (default 10%)
  4. Huber loss for prediction: robust to outliers in industrial KPI data
  5. Auto-train mode: Automatically stops training when R² > 0.85 threshold is
  met
